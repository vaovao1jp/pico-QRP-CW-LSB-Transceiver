# pico_40m_CW_LSB_v1_1J

40m バンド CW/LSB デュアルモード トランシーバー（Raspberry Pi Pico 向け）

---

## 概要

Raspberry Pi Pico2（RP2350）のデュアルコアと内蔵 ADC/PWM を活用した、7MHz（40m）帯の自作トランシーバー用ファームウェアです。CW（モールス）と LSB（下側波帯 SSB）の送受信を 1 台で切り替えて運用できます。受信は SDR（ソフトウェア無線）方式で、OLED にバンドスコープ・ウォーターフォール・波形表示を行います。

- 設計者コールサイン: **JR3XNW**
- 対象 MCU: Raspberry Pi Pico2（RP2350, デュアルコア Cortex-M0+）
- 表示器: SSD1306 128×64 OLED（I2C）
- 周波数生成: Si5351（I2C, 3 出力 CLK0/CLK1/CLK2）

---

## 主な機能

| 機能 | 説明 |
|------|------|
| CW 受信 | IQ ミキサー + ヒルベルト変換 + 狭帯域 IIR BPF（700Hz 中心 ±100Hz） |
| CW 送信 | エレキー / 縦振り電鍵対応、Si5351 CLK2 の ON/OFF でキーイング |
| CW 自動解読 | 受信した CW を文字に変換し画面下部に表示 |
| LSB 受信 | Tayloe 検波器（IQ ミキサー）+ HPF 500Hz（2 次）+ 4 次バターワース LPF 2400Hz（帯域 500–2400Hz） |
| LSB 送信 | Si5351 位相変調方式（uSDX/QCX-SSB と同原理）、マイク音声を送信 |
| バンドスコープ | IQ-FFT による ±15kHz スペクトラム表示 |
| ウォーターフォール | スペクトラムの時間履歴表示 |
| 波形表示 | LSB 受信時は復調音、LSB 送信時はマイク入力波形を最下段に表示 |
| S メーター | バンドスコープのピークパワーをバー表示 |
| 設定保存 | 周波数・ステップ・WPM・キーヤーモード・音量・動作モードを EEPROM に保存 |

### ピン配置

| GPIO | 用途 |
|------|------|
| 0, 1 | ロータリーエンコーダー A/B |
| 2 | STEP_BUTTON（周波数ステップ / 長押しで WPM 設定） |
| 3 | KEY_MODE_BUTTON（キーヤーモード切替 / 長押しで音量） |
| 6 | PADDLE_DOT（ドットパドル, CW） |
| 7 | PADDLE_DASH（ダッシュパドル / 縦振り電鍵, CW） |
| 8 | MODE_BUTTON（CW/LSB モード切替） |
| 9 | PTT_BUTTON（LSB プッシュトーク） |
| 15 | RX_SW（送受信切替, High=TX） |
| 16 | speakerPin（PWM 音声出力, 44100Hz） |
| 17 | PA_BIAS_PIN（PA バイアス制御 PWM, LSB TX 振幅制御） |
| 25 | LED_INDICATOR（処理インジケーター） |
| 26 | I-ch アナログ入力（ADC0） |
| 27 | Q-ch アナログ入力（ADC1） |
| 28 | MIC_IN（マイク入力, ADC2, LSB TX 用） |

### 周辺回路の前提

- **受信フロントエンド**: Tayloe 検波器（IQ ミキサー）で I/Q ベースバンド信号を生成し、GPIO26/27 に入力。
- **Si5351**: CLK0/CLK1 を受信用 LO（90 度位相差）、CLK2 を送信キャリアに使用。
- **PA（パワーアンプ）**: GPIO17 の PWM をゲートバイアスに使い、LSB 送信時に音声振幅へ応じた振幅制御（キャリア抑圧 SSB）を行う。
- **マイク**: エレクトレットマイク等を GPIO28 に接続（出力が微弱なため `MIC_GAIN` で調整）。

## ソフトウェア構成（デュアルコア）

本ファームウェアは RP2040 の 2 コアを役割分担しています。

### Core0（`loop()`）— UI・制御・I2C 一元管理
- ボタン入力処理、モード切替、周波数設定
- FFT 計算とバンドスコープ／ウォーターフォール／波形描画
- **すべての I2C アクセス（OLED 描画 + Si5351 周波数更新）を一元管理**

### Core1（`loop1()`）— オーディオ・DSP
- 40kHz で I/Q を ADC サンプリング → フィルタ → AGC → PWM 音声出力（受信）
- LSB 送信時は 4kHz でマイクをサンプリングし、ヒルベルト変換で瞬時周波数を算出
- **I2C は一切呼ばない**。送信周波数は `txFreqShared` 経由で Core0 に渡す

> **設計上の重要点**: 当初は Core1 から直接 Si5351 を I2C 制御していましたが、`Wire` が Core0 で初期化されているため不安定で、送信後にバンド内ノイズが残る原因になっていました。Core1 から I2C を完全に撤去し、Core0 に集約することで解決しています。コア間共有変数には `volatile` が必須です（特に `transmitting`）。

## 信号処理の詳細

### CW 受信
- IQ ヒルベルト変換（Q を 14 サンプル遅延）で 700Hz 付近の 90 度位相差を生成
- `I − Q_delayed` で LSB 側を選択
- 2 段直列 IIR 2 次 BPF（700Hz 中心 ±100Hz）。Si5351 の周波数誤差（±10ppm ≒ ±70Hz）に耐えるため帯域をやや広めに設定

### LSB 受信
- Tayloe 検波器の I/Q は全音声周波数で 90 度位相差を持つため、`I − Q` の直接減算で LSB を選択
- `lsbQGain`（Core0 が FFT フレームから算出する I/Q 振幅比）で Q を補正し image rejection を改善
- ソフトクリッパー → HPF 500Hz（2 次）→ LPF 2400Hz（4 次）。高域側は 24dB/oct の急峻な減衰で帯域外ノイズを抑制

### LSB 送信（uSDX/QCX-SSB 方式）
1. GPIO28 のマイクを 4kHz でサンプリング、DC オフセット除去・ゲイン適用・ソフトクリップ
2. 15 タップ FIR ヒルベルト変換で I/Q を生成
3. `atan2` で瞬時位相を求め、位相差 → 瞬時周波数偏移 [Hz] に変換
4. 目標周波数を `txFreqShared` に書き込み、Core0 が Si5351 CLK2 を 250µs ごとに更新
5. 同時に I/Q ベクトルの大きさ（alpha-max plus beta-min 近似）から PA バイアス PWM を制御し、キャリア抑圧 SSB を実現

### AGC
- CW と LSB で独立したゲイン変数・パラメータを使用（モード切替時の過大ゲインを防止）
- CW: maxGain=40, 速いアタック・遅いディケイ
- LSB: maxGain=10, 速い応答（強信号での ADC 飽和歪みを防止）

## 操作方法

| 操作 | 動作 |
|------|------|
| ロータリーエンコーダー回転 | 周波数を STEP 単位で上下（7.000–7.200MHz） |
| MODE_BUTTON 短押し | CW ⇔ LSB 切替（同じ局が常に 700Hz で聞こえるよう FREQ を ±700Hz 補正） |
| STEP_BUTTON 短押し | 周波数ステップ切替（1kHz → 100Hz → 10Hz） |
| STEP_BUTTON 長押し | WPM 設定モード |
| KEY_MODE_BUTTON 短押し（CW 時） | エレキー ⇔ 縦振り電鍵 切替 |
| KEY_MODE_BUTTON 長押し | 音量設定モード（×1.0〜×3.0） |
| パドル（CW 時） | CW 送信 |
| PTT_BUTTON（LSB 時） | 押している間 LSB 送信（プッシュトーク） |

## 主要な調整パラメータ

ほとんどの定数は `.ino` 先頭の `#define` にあり、実機に合わせて調整します（`WAVE_SCALE` のみ
`displayLsbWaveform()` 関数内のローカル定数）。現在ソースに設定されている値を「現在値」として示します。

| 定数 | 現在値 | 説明 |
|------|--------|------|
| `MIC_GAIN` | 10.0f | マイク入力ゲイン倍率（マイク回路に合わせる） |
| `PA_TX_DRIVE` | 2 | PA 駆動力 0–8（大きいほど高出力、要 PA 保護） |
| `PA_BIAS_SCALE` | 2.5f | 振幅→PWM デューティのスケール（PA 回路に合わせる） |
| `CW_PA_BIAS_LEVEL` | 4095 | CW 送信時の PA バイアス（フルバイアス固定） |
| `LSB_AGC_MAX_GAIN` | 10.0f | LSB 受信 AGC 最大ゲイン |
| `CW_DETECT_THRESHOLD` | 1.5f | CW 自動解読の検出感度（SNR しきい値） |
| `SCOPE_SPAN_HZ` | 15000.0f | バンドスコープ横軸の表示範囲（±Hz） |
| `SCOPE_SENSITIVITY` | 16.0f | バンドスコープ／ウォーターフォール 振幅ゲイン |
| `SCOPE_OFFSET` | 3.0f | バンドスコープ／ウォーターフォール ノイズフロア持ち上げ量 |
| `WAVE_SCALE`（関数内） | 8.0f | 音声波形表示の感度 |

> ⚠ **送信パワー系の定数（`PA_TX_DRIVE`, `PA_BIAS_SCALE`）は PA の定格を超えないよう、実機で歪みを確認しながら少しずつ上げてください。**

### バンドスコープ／ウォーターフォールの描画調整

バンドスコープとウォーターフォールは同じ振幅データ（`waterfallHistory`）を共有しているため、
下記 3 つを変えると両方の見え方が同時に変化します。

- **`SCOPE_SPAN_HZ`**（既定 15000.0f）: 横軸に表示する周波数範囲（±Hz）。小さくすると拡大表示になり、
  近接信号を見分けやすくなる。大きくするとバンド全体を俯瞰できる。
- **`SCOPE_SENSITIVITY`**（既定 16.0f）: 振幅方向のゲイン。大きくすると弱い信号も高く描画されるが、
  ノイズフロアやウォーターフォールの地色も濃くなる。信号が振り切れる/低すぎる場合に増減する。
- **`SCOPE_OFFSET`**（既定 3.0f）: ノイズフロアの底上げ量。`barLength()` で `log10(振幅)+OFFSET` として
  使われる。小さくするとノイズフロアの表示が下がり、ウォーターフォールの背景ノイズが減る。
- **`THRESHOLD`**（既定 2, `#define`）: ウォーターフォールでドットを打つ振幅しきい値。大きくすると
  弱い信号が消え背景がすっきりする。小さくすると弱い信号も写るがノイズも増える。
- **`WATERFALL_HEIGHT`**（既定 21）: ウォーターフォールの縦ピクセル数（時間履歴の長さ）。

### マイク感度（LSB 送信）の調整

- **`MIC_GAIN`**（既定 10.0f）: マイク入力の増幅倍率。マイク回路に合わせて調整する。
  目安: アンプ内蔵マイクモジュール=5〜10、裸のエレクトレットマイク直結=20〜30、感度の低いマイク=40〜50。
  小さすぎると変調が浅く（送信が弱く）、大きすぎると歪む。
- 送信時のマイク波形は画面最下段に表示されるので、適度に振れる値に合わせると分かりやすい。

### PA（パワーアンプ）関係の調整

- **`PA_TX_DRIVE`**（既定 2, 範囲 0–8）: 送信パワーの段数。1 サンプルあたり 2^drive 倍に左シフトされる。
  1–2=QRP（歪み最小）、3–4=低パワー、5–6=高パワー、7–8=最大（PA 保護回路が必須）。
- **`PA_BIAS_SCALE`**（既定 2.5f）: マイク音声の瞬時振幅を PA ゲートバイアス（PWM デューティ 0–4095）へ
  変換するスケール係数。小さいと出力小（PA 保護）、大きいと出力大（定格超過に注意）。実機で歪みを
  見ながら 2.5f 付近から調整する。
- **`CW_PA_BIAS_LEVEL`**（既定 4095）: CW 送信時の固定 PA バイアス。CW では振幅変調が不要なため常時
  フルバイアスにし、RF の ON/OFF は Si5351 CLK2 で行う。GPIO17 未接続なら効果なし。
- **`PA_BIAS_ENV_ATTACK` / `PA_BIAS_ENV_DECAY`**: 現在の送信処理は振幅を直接 PWM に変換するため未使用
  （将来エンベロープ平滑を行う場合の参考値として残置）。

### CW 自動解読の感度調整

- **`CW_DETECT_THRESHOLD`**（既定 1.5f）: 受信信号の SNR 比（`cwEnvelope`）がこの値を超えると
  キーダウンと判定する。小さくすると弱い信号も拾うが誤検出（ノイズ）が増える。大きくすると確実だが
  取りこぼしやすい。推奨範囲 1.5〜2.5。

### AGC（受信ゲイン）の調整

- **CW**: `applyAGC()` 内の `maxGain=40`（最大ゲイン）, `targetAmp=0.5`, `attackRate=0.01`, `decayRate=0.001`。
  微弱信号を最大限増幅する設定。
- **LSB**: `LSB_AGC_MAX_GAIN=10`, `LSB_AGC_TARGET=0.35`, `LSB_AGC_ATTACK=0.08`, `LSB_AGC_DECAY=0.002`。
  CW より低い最大ゲインと速いアタックで、強信号での ADC 飽和歪みを防ぐ。
  受信音が小さいときは `maxGain` を上げ、強信号で歪むときは下げる。

### その他の調整に使う定数

- **`WAVE_SCALE`**（`displayLsbWaveform()` 内, 既定 8.0f）: 最下段の音声波形表示の感度。
  3.0=大きな声/強信号でのみ振れる、8.0=通常会話で適度、15.0=微弱信号でも振れる（ノイズも見える）。
  併せて `WAVE_MAX_AMP`（既定 3px）で上下の振れ幅を制限する。
- **`LSB_SOFT_CLIP_THRESH`**（既定 0.7f）: LSB 受信フィルタ入力のソフトクリップしきい値。強信号で
  IIR フィルタが発振するのを防ぐ。歪む場合は下げる。
- **`DEFAULT_WPM`**（既定 20）: キーヤーの初期 WPM（実際の値は EEPROM 保存値が優先）。
- **`LOW_FREQ` / `HI_FREQ`**（既定 7000000 / 7200000）: VFO の可変範囲（Hz）。
- **`CW_TONE` / `CW_AUDIO_OFFSET`**（既定 700Hz / 70000centihz）: CW のモニター音とビート周波数。
  両者を揃えて変更すると CW のピッチを変えられる。

## ビルド方法

### 必要な開発環境
- Arduino IDE（または arduino-cli）
- **arduino-pico** ボードパッケージ（Earle Philhower 版, Raspberry Pi Pico 用）

### 必要なライブラリ
| ライブラリ | 入手先 |
|-----------|--------|
| Rotary | https://github.com/brianlow/Rotary |
| U8g2 | https://github.com/olikraus/U8g2_Arduino |
| arduinoFFT | arduinoFFT v2.0.4 |
| Si5351 (Etherkit) | https://github.com/etherkit/Si5351Arduino |
| EEPROM / Wire | Pico ボードパッケージ内蔵 |

### 手順
1. Arduino IDE に arduino-pico をインストール
2. 上記ライブラリをライブラリマネージャ等で導入
3. ボードに「Raspberry Pi Pico2」を選択
4. `pico_40m_CW_LSB_v1_1J.ino` を開いて書き込み
